# 3. 项目管理计划 (2周冲刺版)

## 1. 项目概况
*   **项目名称**：智搜研报助手 (AI Research Assistant Lite)
*   **周期**：2周 (14天)
*   **目标**：交付一个可运行的 MVP，具备联网搜索和 PDF 导出功能。

## 2. 里程碑计划

| 阶段 | 时间 | 核心任务 | 状态 |
| :--- | :--- | :--- | :--- |
| **Week 1: 核心功能** | Day 1-3 | 环境搭建，Hello World (Streamlit + FastAPI) | ✅ 完成 |
| | Day 4-5 | LangGraph 基础链路跑通 (Chatbot Node) | ✅ 完成 |
| | Day 6-7 | 接入 Tavily 搜索工具，实现 Agent 路由逻辑 | ✅ 完成 |
| **Week 2: 增强与交付** | Day 8-9 | 实现流式输出 (Streaming)，优化前端体验 | ✅ 完成 |
| | Day 10-11 | 开发 PDF 导出功能 (ReportLab)，解决中文乱码 | ✅ 完成 |
| | Day 12 | 多模型适配 (OpenAI/Aliyun/DeepSeek)，UI 美化 | ✅ 完成 |
| | Day 13 | 集成测试，修复 Bug (PDF 空白、字体路径等) | ✅ 完成 |
| | Day 14 | 编写文档，准备演示材料 | ✅ 完成 |

## 3. 风险管理

| 风险点 | 可能性 | 影响 | 应对措施 |
| :--- | :--- | :--- | :--- |
| **API 额度耗尽** | 高 | 无法搜索 | 申请多个 Key；允许用户自定义 Key；后端增加错误提示。 |
| **中文 PDF 乱码** | 高 | 报告不可用 | 提前准备 `simsun.ttc` 字体文件并测试加载路径。 |
| **LangGraph 状态丢失** | 中 | 体验中断 | Lite 版采用内存存储，明确告知用户刷新会丢失历史；PDF 导出改为由前端传入数据以解耦。 |
| **依赖安装失败** | 中 | 无法运行 | 提供详细的 `requirements.txt` 和安装文档；使用国内镜像源。 |

## 4. 交付物清单
1.  **源代码**：Frontend + Backend。
2.  **可执行环境**：`requirements.txt`。
3.  **文档**：
    *   需求规格说明书
    *   系统设计文档
    *   接口设计说明书
    *   用户手册 (集成在 README 中)
