# 1. 项目提案/商业计划书 (精简版)

## 1.1 执行摘要

在学术研究和课程作业中，学生常面临“信息搜集耗时”和“报告格式繁琐”两大痛点。通用大模型（LLM）虽能对话，但缺乏最新的联网信息，且无法直接生成提交级别的PDF文档。

本项目旨在**2周**内快速构建一个轻量级的**“智搜研报助手”**（Lite Research Agent）。该系统基于**LangGraph**技术，专注于解决学生群体的核心需求：智能联网搜索最新资料，并将对话整理为标准的研报PDF。我们采用**Streamlit**进行快速原型开发，以最小的成本实现“所见即所得”的交付体验。

## 1.2 项目背景与目标

### 1.2.1 项目背景
作为软件工程专业的学生，我们在完成课程设计时，需要查阅大量最新文献和技术博客，随后还要花费大量时间排版Word/PDF。现有的AI工具（如Kimi、Perplexity）虽然强大，但往往需要付费订阅，且导出功能有限。参考清华“数据派THU”关于Agent技术的趋势，我们决定利用LangGraph构建一个属于学生自己的开源研报助手。

### 1.2.2 项目目标
1.  **核心功能**：实现一个具备“思考-搜索-回答”能力的Agent。
2.  **敏捷开发**：利用Python生态（LangGraph + Streamlit）在短时间内完成全栈开发。
3.  **实用交付**：支持将对话记录一键导出为带引用来源的PDF，满足作业提交需求。

## 1.3 市场分析

### 1.3.1 目标用户
*   **在校大学生/研究生**：需要写论文综述、课程报告。
*   **个人开发者**：需要快速调研新技术栈。

### 1.3.2 痛点与解决方案
| 痛点 | 解决方案 |
| :--- | :--- |
| LLM知识过时 | 集成Tavily Search API，获取实时网页摘要。 |
| 复制粘贴繁琐 | 内置PDF生成引擎，一键生成规范文档。 |
| 商业软件昂贵 | 自研轻量级工具，通过API Key按需付费，成本极低。 |

## 1.4 产品功能

### 1.4.1 最小可行性产品 (MVP)
1.  **智能搜索对话**：用户提问 -> Agent判断是否联网 -> 搜索并汇总信息 -> 回答。
2.  **来源标注**：在回答末尾列出参考网页链接。
3.  **PDF导出**：点击按钮，下载包含当前对话内容的PDF文件。

### 1.4.2 技术亮点
*   **LangGraph**：使用图结构编排Agent逻辑，相比传统Chain结构更灵活，易于调试。
*   **Streamlit**：纯Python编写前端，极大地缩短了UI开发周期。

## 1.5 预期成本与收益

### 1.5.1 预期成本
*   **人力**：2人 x 2周。
*   **资金**：API调用费用 < 50元人民币（开发调试期）。
*   **设施**：个人笔记本电脑本地运行。

### 1.5.2 预期收益
*   **技能收益**：熟练掌握LangGraph框架和Streamlit快速开发模式。
*   **课程收益**：提交一个运行稳定、逻辑清晰的课程设计作品，争取高分。

## 1.6 实施计划 (2周冲刺)
*   **第1-3天**：需求确认、技术选型、环境搭建。
*   **第4-8天**：LangGraph核心逻辑开发、搜索API对接。
*   **第9-12天**：Streamlit界面开发、PDF导出功能实现。
*   **第13-14天**：集成测试、Bug修复、文档完善与答辩准备。
