# 2-3. 专家评审意见及整改结果 (精简版)

## 1. 评审会议概况
*   **会议时间**：2025年12月1日
*   **评审专家**：秦勇老师（组长）
*   **评审重点**：项目可行性、范围控制、技术选型

## 2. 评审意见汇总

针对“2周敏捷开发”的特殊情况，专家组提出了以下关键建议：

| 序号 | 类别 | 问题描述 | 提出人 |
| :--- | :--- | :--- | :--- |
| 1 | 范围控制 | 原计划的“多用户登录”和“复杂权限”在2周内难以高质量完成，建议砍掉。 | 秦勇老师 |
| 2 | 技术选型 | 前端使用React开发周期过长，建议改用Python生态的Streamlit或Gradio。 | 秦勇老师 |
| 3 | 考核重点 | 考核的核心在于LangGraph的逻辑流转和Tavily搜索的集成，界面UI只需整洁即可。 | 秦勇老师 |

## 3. 整改结果

项目组完全接受专家建议，并进行了如下调整：

### 3.1 范围调整
*   **整改措施**：删除了所有与用户注册、登录、找回密码相关的功能。系统改为单机版/单Session版。
*   **状态**：已在《需求规格说明书》中更新。

### 3.2 技术栈变更
*   **整改措施**：前端从 React + Ant Design 更改为 **Streamlit**。这将开发效率提升了约 3 倍。
*   **状态**：已在《系统设计文档》中更新。

### 3.3 核心功能聚焦
*   **整改措施**：集中精力优化 Agent 的“搜索意图识别”准确率，放弃了“本地知识库RAG”等锦上添花的功能。
*   **状态**：已落实到开发计划中。

## 4. 结论
调整后的方案目标明确，工作量适中，符合2周课程设计的考核要求。
